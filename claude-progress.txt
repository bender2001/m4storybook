    variant that silently fails. Combine

    (e.g. `bg-surface-variant`) for a token-driven disabled

- Re-usable lessons from this slice:
  * When two Tailwind utilities target the same CSS property and
    both end up in the class list (e.g. `flex-col` + `flex-row`),
    source-order in the generated CSS — not className order in
    the markup — decides which wins. Don't bake the "default"
    direction into the base anatomy if the component might swap
    it; pick one direction in a ternary so only one utility lands.
  * For groups with a legend + helper text, wire legend to
    aria-labelledby and helper text to aria-describedby. They
    play different a11y roles and conflating them confuses
    screen readers.
  * useId-generated ids contain ":" characters which are invalid
    in CSS selectors. In Playwright tests, look up by attribute
    value (`page.locator('[id="..."]')`) rather than `#id` when
    the id was generated by React's useId.
  * For toggleable visual containers that wrap an input + state-
    layer + visual circle, the Checkbox pattern transfers cleanly
    to Radio: outer `<label htmlFor>`, inner `<div data-radio-
    control>` with absolutely-positioned input, and a `<span
    data-state-layer>` painted under the visual element.
