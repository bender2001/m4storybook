    via `useEffect` whenever the value prop changes.

    indeterminate state; assert it in tests rather than reading
    the DOM `indeterminate` property.

    (it's in the generated CSS); it's the `bg-{role}/[0.38]`
    variant that silently fails. Combine

    (e.g. `bg-surface-variant`) for a token-driven disabled
    treatment that actually renders.

- Re-usable lessons from this slice:
  * For inputs that have a strong native HTML semantic

- Reusable lessons from this slice:
  * `min-h-8`, `min-h-12`, `min-h-14` are NOT in
    Tailwind v3's default `minHeight` scale —
    Tailwind ships `min-h-{0, full, screen, min,
    max, fit, svh, lvh, dvh}` and the integer
    spacing values stop at `min-h-px`/`min-h-0`.
    `min-h-8` resolves to `min-height: auto`,
    not 32px. Use the arbitrary syntax
    `min-h-[32px]` when you need a token-shell
    height. The same caveat applies to `h-` and
    `w-` for non-default spacing values.
  * `motion/react` writes the active animation
    state (opacity, height, transform) to the
    target element as INLINE styles — those
    inline styles BEAT Tailwind classes by CSS
    specificity. So `disabled` cannot apply a
    CSS class like `opacity-[0.38]` if the
    motion `animate` prop is also setting
    opacity. The fix is to push the disabled
    opacity through the `animate` prop itself
    (e.g. `animate={{ opacity: disabled ?
    0.38 : 1 }}`) so motion drives both states.
    Cursor + pointer-events still work as CSS
    classes because motion doesn't touch them.

    motion.div whose `initial={{ opacity: 0,
    height: 0 }}` lets the alert mount without
    animating in but still animate OUT via the
    `exit` prop when the open prop flips. This
    matches the M3 banner enter/exit guidance
    (no entrance flash on first paint).
  * Tailwind's opacity modifier `bg-current/10`
    silently fails on `currentColor` because
    Tailwind needs RGB triples to slot into
    `rgb(... / opacity)`. Don't reach for it on
    a close-button hover wash — use an opacity
    transition on the icon itself instead
    (`opacity-80 hover:opacity-100`). Same
    constraint as the var()-color tokens
    documented in the auto-memory.
